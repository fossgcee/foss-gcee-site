# Contributing to FOSS GCEE

Thanks for taking the time to improve this project. Contributions from first-time and experienced contributors are both welcome.

## How to Get Started

1. Fork or branch from the main repository.
2. Install dependencies.
3. Run the app locally and make your change.
4. Test the change before opening a pull request.

## Local Setup

This project uses Next.js and standard npm scripts.

```bash
npm install
npm run dev
```

Useful scripts:

```bash
npm run build
npm run lint
```

## Before You Submit

- Check whether your issue or idea already exists.
- Use the issue templates in `.github/ISSUE_TEMPLATE/` when reporting bugs, suggesting features, requesting documentation updates, or sharing ideas for new features.
- Use the pull request template in `.github/pull_request_template.md`.
- Keep changes focused and easy to review.

## Bug Reports

When reporting a bug, please include:

- System info such as OS and kernel
- Steps to reproduce
- Expected behavior
- Actual behavior
- Any relevant screenshots or logs

## Feature Suggestions

When suggesting a feature or new idea, please include:

- The problem you are trying to solve
- Why the feature would help
- Any expected behavior or user flow
- Any examples, mockups, or references

## Documentation Updates

When requesting a docs change, please include:

- The page or section that needs attention
- What is inaccurate, missing, or confusing
- A suggested correction if you have one

## Pull Requests

Please include a short summary of your change, testing notes, and screenshots if the update affects the UI.

## Style Guide

- Match the existing code style and project patterns.
- Prefer small, incremental changes.
- Keep language clear and friendly in user-facing copy.

## Need Help?

If you are unsure where to start, open a discussion or issue and describe what you are trying to do. We will help you figure out the next step.
